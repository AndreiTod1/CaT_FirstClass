GreenCamping - HTML Documentation

Documentație pentru platforma GreenCamping, o aplicație web pentru descoperirea, rezervarea și administrarea campingurilor din România.

DEPLOY

1. Introducere

GreenCamping este o platformă web pentru descoperirea, rezervarea și administrarea campingurilor din România. Utilizatorii pot explora locații, pot face rezervări, pot lăsa recenzii și pot interacționa cu o comunitate de pasionați de camping. Platforma oferă și un panou de administrare pentru gestionarea campingurilor, utilizatorilor și rezervărilor.

2. Funcționalități principale

  • Autentificare cu Google OAuth
  • Listare și filtrare campinguri după regiune, facilități, preț, rating
  • Vizualizare detalii camping (descriere, galerie foto, facilități, recenzii)
  • Rezervare camping cu selectare dată și număr de persoane
  • Adăugare și vizualizare recenzii cu rating și imagini
  • Hartă interactivă cu locațiile campingurilor

3. Funcționalități Admin

  • Adăugare, editare și ștergere campinguri
  • Gestionare utilizatori (roluri, ștergere, modificare)
  • Vizualizare și ștergere rezervări
  • Statistici: număr campinguri, utilizatori, rezervări, rating mediu
  • Filtrare și căutare rapidă în panoul de administrare

4. Model de date (PostgreSQL)

  • users: id, email, password_hash, name, role, created_at
  • campgrounds: id, name, region, type, price, latitude, longitude, description, status, created_at
  • bookings: id, user_id, camp_site_id, start_date, end_date, guests, status, created_at
  • reviews: id, user_id, camp_site_id, rating, comment, created_at
  • media: id, review_id, url, type

Relațiile sunt gestionate prin chei externe (user_id, camp_site_id, review_id).

5. Interfață utilizator

  1. Utilizatorul accesează pagina principală.
  2. Poate naviga la camping-uri sau hartă. pentru a vedea locațiile campingurilor.
  3. Pentru a vizualiza detalii, poate accesa pagina de detalii a unui camping.
  4. După autentificare, poate face rezervări și lăsa recenzii pe pagina camping ului respectiv.
  5. Utilizatorii cu rol de admin accesează panoul de administrare pentru gestionare completă.

Securitate

  • Autentificare cu JWT, stocat în cookie HttpOnly
  • Parolele utilizatorilor nu sunt stocate local
  • Validare input pe backend și frontend
  • Protecție prin verificare roluri
  • Acces la panoul de admin doar pentru utilizatori cu rol "admin"
  • Upload fișiere cu limită de dimensiune și validare extensie

Cerințe

  • Node.js 18+
  • PostgreSQL 13+
  • Browser modern (Chrome, Firefox, Edge, Safari)
  • Conexiune la internet pentru Google OAuth și hărți Leaflet

Set-up

Enviroment variables:

  • GOOGLE_SECRET
  • GOOGLE_REDIRECT
  • GOOGLE_ID
  • PORT
  • PG_CONN=conexiune PostgreSQL
  • JWT_SECRET=superSecretKey
